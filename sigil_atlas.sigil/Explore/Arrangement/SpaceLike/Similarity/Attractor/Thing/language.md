# Thing

Anything that I can name. Flower. Person. Green jacket. Bug.

It is, of course, a @sigil: #named and !recognizable.

A @thing can be primitive — a leaf name — or compound. A compound @thing is a decision tree: each constituent is a discrimination step refining which leaf applies. Beetles, butterflies, flies, dragonflies → insects; insects, spiders, ... → bugs. Each #compose goes one level up in abstraction.

An @image satisfies a compound @thing if it satisfies any leaf reachable by descent. The tree is embedded structurally, not collapsed to a flat name: evaluation against a @visionModel is iterative — each frame resolves one level of distinction ("is this a bug?" then "is this an insect?" then "is this a beetle?"). The depth descended matches the @resolution needed to display. Zoomed out, coarse distinctions suffice; zoomed in, descent continues until the last discriminating degree.

The @taxonomy is scaffolding, not a gate. Any name I type becomes a @thing; when the name matches a @taxonomy term, its canonical prompt enriches the query, but unmatched names are equally valid. The language must not constrain the story — if I am reasonably good at telling it, the listener should see it.

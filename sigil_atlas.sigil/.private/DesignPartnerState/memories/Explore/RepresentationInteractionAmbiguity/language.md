A major source of decoherence in the spec for @Explore is ambiguity between representation and interaction. Terms such as neighborhood, slice, timelike direction, relevance, zoom generation, and preference currently mix experiential meanings with implementation meanings. This ambiguity means multiple incompatible systems could all appear faithful to @Explore. To keep code reprojectable from @Explore, each such term needs one canonical meaning, one owned state location, and one precise update rule.
